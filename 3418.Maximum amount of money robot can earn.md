# 🧮 Maximum Coins Robot Path Problem

## 🚀 Problem Statement

You are given an **m × n grid** where a robot starts at the **top-left
corner (0,0)** and wants to reach the **bottom-right corner (m-1,n-1)**.

### 👉 Movement Rules:

-   The robot can only move:
    -   **Right ➡️**
    -   **Down ⬇️**

------------------------------------------------------------------------

## 💰 Cell Values

Each cell contains a value `coins[i][j]`:

-   If `coins[i][j] >= 0` → Robot **gains coins**
-   If `coins[i][j] < 0` → Robot encounters a **robber** and **loses
    coins**

------------------------------------------------------------------------

## 🛡️ Special Ability

-   The robot can **neutralize robbers in at most 2 cells**
-   This means:
    -   It can ignore the negative effect in **up to 2 cells**

------------------------------------------------------------------------

## 🎯 Goal

Return the **maximum profit (coins)** the robot can collect.

> ⚠️ Note: The total coins can be negative.

------------------------------------------------------------------------

## 🧠 Approach (Dynamic Programming)

We use a **3D DP** approach:

### 🔹 State Definition:

    dp[i][j][k] = maximum coins at cell (i, j) using k neutralizations

Where: - `i, j` → current position - `k` → number of robbers neutralized
(0 to 2)

------------------------------------------------------------------------

## ⚙️ Transition

At each cell: - Move from: - Top `(i-1, j)` - Left `(i, j-1)`

### Case 1: Normal cell (≥ 0)

    dp[i][j][k] = max(prev) + coins[i][j]

### Case 2: Robber (\< 0)

-   Take loss:

```{=html}
<!-- -->
```
    dp[i][j][k] = max(prev) + coins[i][j]

-   OR neutralize (if k \> 0):

```{=html}
<!-- -->
```
    dp[i][j][k] = max(prev with k-1)

------------------------------------------------------------------------

## 🧾 Python Solution

``` python
class Solution:
    def maximumAmount(self, coins):
        m, n = len(coins), len(coins[0])

        # dp[i][j][k]: max coins at (i,j) with k neutralizations used
        dp = [[[-float('inf')] * 3 for _ in range(n)] for _ in range(m)]

        # Start position
        for k in range(3):
            if coins[0][0] >= 0:
                dp[0][0][k] = coins[0][0]
            else:
                if k > 0:
                    dp[0][0][k] = 0  # neutralize
                else:
                    dp[0][0][k] = coins[0][0]

        for i in range(m):
            for j in range(n):
                for k in range(3):
                    if i == 0 and j == 0:
                        continue

                    best = -float('inf')

                    # from top
                    if i > 0:
                        best = max(best, dp[i-1][j][k])

                    # from left
                    if j > 0:
                        best = max(best, dp[i][j-1][k])

                    if coins[i][j] >= 0:
                        dp[i][j][k] = best + coins[i][j]
                    else:
                        # take loss
                        dp[i][j][k] = best + coins[i][j]

                        # neutralize
                        if k > 0:
                            best_prev = -float('inf')
                            if i > 0:
                                best_prev = max(best_prev, dp[i-1][j][k-1])
                            if j > 0:
                                best_prev = max(best_prev, dp[i][j-1][k-1])

                            dp[i][j][k] = max(dp[i][j][k], best_prev)

        return max(dp[m-1][n-1])
```

------------------------------------------------------------------------

## 📊 Example 1

    Input:
    coins = [[0,1,-1],
             [1,-2,3],
             [2,-3,4]]

    Output: 8

------------------------------------------------------------------------

## 📊 Example 2

    Input:
    coins = [[10,10,10],
             [10,10,10]]

    Output: 40

------------------------------------------------------------------------

## ⏱️ Complexity

  Type    Complexity
  ------- --------------
  Time    O(m × n × 3)
  Space   O(m × n × 3)

------------------------------------------------------------------------

## 💡 Key Insight

-   Think of this as a **grid DP with limited "power-ups"**
-   You must decide **where to use neutralization strategically**

------------------------------------------------------------------------

## 🎉 Summary

-   Use DP with 3 states (0,1,2 neutralizations)
-   At each robber cell, choose:
    -   Take loss ❌
    -   Neutralize ✅
-   Return the best possible result
