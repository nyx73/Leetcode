# Equal Sum Partition of Grid

## 🧩 Problem

You are given an `m × n` matrix `grid` of positive integers.

Your task is to check if you can make:
- **one horizontal cut** OR
- **one vertical cut**

such that:

### ✅ Conditions
1. Both resulting sections are **non-empty**
2. The **sum of elements** in both sections is **equal**

Return `true` if possible, otherwise `false`.

---

## 💡 Key Insight

Let:
```
total_sum = sum of all elements in grid
```
👉 If `total_sum` is **odd**, it's impossible to split into two equal parts.

So:

if total_sum % 2 != 0 → return False


---

## 🧠 Approach

We check both possibilities:

---

### 🔹 1. Horizontal Cut

- Compute row-wise sums
- Keep adding rows from top
- Check if cumulative sum becomes `total_sum / 2`

---

### 🔹 2. Vertical Cut

- Compute column-wise sums
- Keep adding columns from left
- Check if cumulative sum becomes `total_sum / 2`

---

## ✅ Python Solution (LeetCode Ready)

```python
class Solution:
    def canPartitionGrid(self, grid):
        m, n = len(grid), len(grid[0])
        
        total_sum = sum(sum(row) for row in grid)
        
        # If total sum is odd → impossible
        if total_sum % 2 != 0:
            return False
        
        target = total_sum // 2
        
        # 🔹 Check horizontal cuts
        curr_sum = 0
        for i in range(m - 1):  # ensure bottom part is non-empty
            curr_sum += sum(grid[i])
            if curr_sum == target:
                return True
        
        # 🔹 Check vertical cuts
        curr_sum = 0
        for j in range(n - 1):  # ensure right part is non-empty
            col_sum = 0
            for i in range(m):
                col_sum += grid[i][j]
            
            curr_sum += col_sum
            if curr_sum == target:
                return True
        
        return False
```

🧪 Example 1
Input:
grid = [[1,4],
        [2,3]]
Explanation:
Total = 10 → target = 5
Horizontal cut:
Top = 5
Bottom = 5 ✅
Output:
True
🧪 Example 2
Input:
grid = [[1,3],
        [2,4]]
Explanation:
Total = 10 → target = 5
No valid cut exists ❌
Output:
False
⏱️ Complexity
Type	Complexity
Time	O(m × n)
Space	O(1)
🚀 Pro Tip

This is a classic pattern:

🔁 "Partition into equal sum" → Check prefix sums

Used in:

Array partition problems
Grid cut problems
Subarray sum problems
🔥 Optimization Idea

To optimize vertical cuts:

Precompute column sums once
Avoid recomputing inside loop
