# Matrix Cyclic Shift Problem

## 🧩 Problem Statement
You are given an `m x n` integer matrix `mat` and an integer `k`.

The following process happens `k` times:

- Even-indexed rows (0, 2, 4, ...) are cyclically shifted to the **left**.
- Odd-indexed rows (1, 3, 5, ...) are cyclically shifted to the **right**.

Return **true** if the final modified matrix after `k` steps is identical to the original matrix, otherwise return **false**.

---

## 💡 Key Insight

Instead of simulating `k` steps one by one:

- A left or right cyclic shift repeated `k` times is equivalent to shifting by `k % n`.
- If after applying this net shift, every row returns to its original form → return `True`.

---

## 🚀 Approach

1. Let `n = number of columns`
2. Compute `k = k % n`
3. For each row:
   - If even index → left shift by `k`
   - If odd index → right shift by `k`
4. Compare with original row

---

## 🧠 Python Solution

```python
class Solution:
    def areSimilar(self, mat, k):
        m, n = len(mat), len(mat[0])
        k %= n

        for i in range(m):
            if i % 2 == 0:
                # Left shift
                shifted = mat[i][k:] + mat[i][:k]
            else:
                # Right shift
                shifted = mat[i][-k:] + mat[i][:-k]

            if shifted != mat[i]:
                return False

        return True
```

---

## 🧪 Example

### Input:
```
mat = [[1,2,1,2],[5,5,5,5],[6,3,6,3]]
k = 2
```

### Output:
```
True
```

---

## ⏱ Complexity

- **Time Complexity:** O(m × n)
- **Space Complexity:** O(n) (for shifted row)

---

## ✨ Notes

- If all elements in a row are the same → always remains unchanged.
- If `k % n == 0` → matrix always remains the same.
