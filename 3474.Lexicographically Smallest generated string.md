# 🧠 Problem Statement

You are given two strings:

- str1 of length n (consisting of 'T' and 'F')
- str2 of length m

You need to construct a string word of length n + m - 1 such that:

✅ Rules:

For every index i (0 ≤ i < n):

- If str1[i] == 'T'
→ word[i ... i+m-1] == str2
- If str1[i] == 'F'
→ word[i ... i+m-1] != str2

--------------

# 🎯 Goal

- Return the lexicographically smallest valid string word
- If impossible → return ""

------------

# 💡 Approach

**Step 1**: Initialize Result
 
- Create a result array res of size n + m - 1
- Fill it with '?' (unknown characters)

**Step 2** : Apply 'T' Constraints

- For each index i where str1[i] == 'T':

Force:

res[i + j] = str2[j]
- If conflict occurs → return ""

**Step 3** : Replace Remaining '?'
- Replace all '?' with 'a' (smallest lexicographic character)

**Step 4**: Validate 'F' Constraints

- For each index i where str1[i] == 'F':
- Check substring:
res[i ... i+m-1]
- If it equals str2 → invalid → return ""

-------------------

# ⚙️ Python Implementation

```
class Solution:
    def generateString(self, str1: str, str2: str) -> str:
        n, m = len(str1), len(str2)
        res = ['?'] * (n + m - 1)

        # Step 1: Apply 'T' constraints
        for i in range(n):
            if str1[i] == 'T':
                for j in range(m):
                    if res[i + j] == '?' or res[i + j] == str2[j]:
                        res[i + j] = str2[j]
                    else:
                        return ""

        # Step 2: Fill remaining '?' with 'a'
        for i in range(len(res)):
            if res[i] == '?':
                res[i] = 'a'

        # Step 3: Validate 'F' constraints
        for i in range(n):
            if str1[i] == 'F':
                if ''.join(res[i:i + m]) == str2:
                    return ""

        return ''.join(res)
```

# 📊 Examples

✅ Example 1
**Input:**
str1 = "TFTF"
str2 = "ab"

**Output:**
"ababa"

❌ Example 2
**Input:**
str1 = "TFTF"
str2 = "abc"

**Output:**
""

✅ Example 3
**Input:**
str1 = "F"
str2 = "d"

**Output:**
"a"

#⚡ Complexity

Type	Complexity
Time	O(n × m)
Space	O(n + m)

----------------------------
# 🔥 Key Insights
- 'T' forces exact placement → strict constraint
- 'F' only checks inequality → flexible
- Greedy + validation works best
- Always prefer 'a' for lexicographically smallest result

----------------------

# 🎉 Summary

- Build the string step-by-step
- Apply strict matches first ('T')
- Fill remaining with smallest characters
- Validate forbidden cases ('F')
