# Flip Square Submatrix Vertically

## 🧩 Problem Statement

You are given:
- An `m x n` integer matrix `grid`
- Three integers `x`, `y`, and `k`

Where:
- `(x, y)` represents the **top-left corner** of a square submatrix
- `k` is the **size (side length)** of the square

### 🎯 Task
Flip the submatrix **vertically**, meaning:
- Reverse the order of its rows
- Keep columns unchanged

Return the updated matrix.

---

## 🧠 Concept & Logic

To flip the submatrix vertically:
- Swap the **top row with the bottom row**
- Swap the **second row with the second-last row**
- Continue this process until the middle is reached

### 🔁 Key Idea

For each row `i` in the first half of the submatrix:
- Swap row `(x + i)` with `(x + k - 1 - i)`
- Only for columns from `y` to `y + k - 1`

---

## 🛠️ Algorithm Steps

1. Loop through half of the rows: `i = 0 → k//2 - 1`
2. For each column `j = 0 → k - 1`
3. Swap:

grid[x + i][y + j] ↔ grid[x + k - 1 - i][y + j]

---

## 💻 Python Implementation

```python
class Solution:
 def reverseSubmatrix(self, grid, x, y, k):
     # Loop through half of the submatrix rows
     for i in range(k // 2):
         for j in range(k):
             # Swap elements vertically
             grid[x + i][y + j], grid[x + k - 1 - i][y + j] = \
             grid[x + k - 1 - i][y + j], grid[x + i][y + j]
     
     return grid
```

✅ Example 1
Input:

]
```

x = 1

Output:

]
```

## ⏱️ Complexity Analysis
Type	Complexity
Time	O(k²)
Space	O(1)

We only traverse the k x k submatrix
No extra space is used (in-place swapping)
##  ⚡ Key Takeaways
Only swap rows, not columns
Operate within the submatrix bounds
Use in-place swapping for efficiency
